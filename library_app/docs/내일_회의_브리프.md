> 목적: 말로 설명하지 말고 **이 한 장 + 데모**로 팀 합의를 받는다.
> 30초 요약 → 두 트랙 그림 → 확정 안건 9개 → 분석 증거 순으로 읽으면 된다.

## 0. 30초 요약 (팀원에게 첫 문장)

> "우리는 예쁜 그래프 대시보드가 아니라, **사서가 도서관을 고르면 '안 사야 할 책 / 더 사야 할 분야'를 통계 근거와 함께 자동으로 짚어주는 장서 진단 도구**를 만든다.
> 핵심 차별점은 **자기참조 수급격차** — 외부 기준이 아니라 *그 도서관 안의 공급 vs 수요*를 본다."

## 1. 우리는 "한 몸통, 두 얼굴"로 간다 (가장 중요한 그림)

같은 정제 데이터에서 **두 산출물**을 동시에 뽑는다. 둘 중 하나만 하면 수업 또는 공모전 한쪽이 빈다.

```
            [한 몸통: 같은 정제 데이터 + 파생변수]
            ┌──────────────────┴──────────────────┐
   ① 공모전 데모 트랙                      ② 수업/분석 트랙
   집계 기반 자동 진단                     로지스틱 회귀(미대출 예측)
   (수급격차 → 처방 카드)                  ("안 빌려지는 책의 조건")
   산출물: 선택형 대시보드                  산출물: 분석 노트북 + 보고서
   상태: 거의 완성(app.py)                 상태: 가능 확인됨(아래 4번)
```

- ①은 "현안을 해결하나, 전국으로 확장되나"(공모전 채점 기준)에 답한다.
- ②는 "분석 과정을 제대로·설명 가능하게 했나"(수업 채점 기준)에 답한다.
- **둘은 데이터를 공유한다.** 분석은 한 번 돌리고, 트랙별로 다르게 뽑는다.

## 2. 1차 MVP 범위 (이번 주에 닫는 것)

| MVP에**넣는다**                     | 2차로**미룬다** (오늘 손대지 않기로 합의) |
| ----------------------------------------- | ----------------------------------------------- |
| 사전 적재 2관(2.28·독산)**선택형** | 도서관정보나루**API 실시간 연동**         |
| 수급격차(분야·독자대상) + 처방 카드      | 전국 1,598관 마스터·**지도/접근성**      |
| **로지스틱 회귀 1개**(미대출 예측)  | **군집** 등 추가 모델                     |
| 카이제곱 1개(격차 유의성)                 | 키워드 워드클라우드, 도서상세                   |
| HTML 보고서 + 배포 URL 1개                | 검증사례 다관 비교 대시보드                     |

> 가드레일: "웹앱 욕심이 분석 시간을 잠식한다." 위 경계를 회의에서 못박지 않으면 1차가 안 닫힌다.

## 3. 회의에서 확정할 9가지 (이걸 다 정하면 끝)

1. **최종 주제명** — 예: "공공도서관 장서 수급 불균형 진단 및 수서 우선순위 추천"
2. **핵심 사용자** — 공공도서관 사서·장서 담당자
3. **핵심 화면 질문 4개** — 더 사야 할 분야? / 복본 필요 분야? / 이용 낮은 장서? / 공급·수요 일치?
4. **필수 컬럼** — 도서명·ISBN·KDC·도서권수·대출건수·발행년도·등록일자
5. **MVP 범위** — 위 2번 표대로 (선택형 + 로지스틱 포함)
6. **API 연동** — 1차에서 **뺀다**(2차). 먼저 사전적재로 엔진 안정화
7. **로지스틱(②트랙) 담당·일정** ⭐ — 수업 점수 핵심. *MVP와 동시에* 간다고 못박기
8. **업무분장 버전 통일** ⭐ — 아래 4번 표로 단일화 (현재 문서 2개가 서로 다름)
9. **수업 제출물 형식** — 노트북 + 보고서 + 발표, 각자 자기 단계의 "왜"를 방어

⭐ = 인수인계 문서에 흐릿해서 우리가 새로 못박는 항목.

### 업무분장 (단일안 — 분석 단계별 소유)

> 각자 = **분석 질문 1개 + 통계 1개 + 산출물 1개**. "코드만 돌렸다"가 안 되게 단계를 통째로 소유.

| 담당                 | 소유 단계                            | 맡는 통계               | 산출물(채점 대상)                                        |
| -------------------- | ------------------------------------ | ----------------------- | -------------------------------------------------------- |
| 위성 A               | 데이터·스키마·전처리               | 기술통계·품질점검      | 표준 스키마 + 컬럼 매핑표 + 정제 데이터                  |
| 위성 B               | EDA·시각화·화면 흐름               | 기술통계·분포          | 분포·회전율 시각화 + 기본분석 화면                      |
| 위성 C               | 수급격차·가설검정                   | 카이제곱                | 격차표 + 유의성 해석                                     |
| **허브(본인)** | **로지스틱·처방·통합·배포** | **로지스틱 회귀** | 미대출 예측 + 수서/복본/정리 추천 + 선택형 대시보드 배포 |

## 4. "②트랙 진짜 되나?" — 실제 데이터로 검증 완료

종속변수 `미대출여부 = 대출건수==0`으로 로지스틱이 **실제로 돌고 의미가 나온다.**

**종속변수 균형 (망하지 않음):**

| 도서관 | 미대출(Y=1) | 대출됨 | 판정                 |
| ------ | ----------- | ------ | -------------------- |
| 2.28   | 34.0%       | 66.0%  | 교과서급 균형        |
| 독산   | 12.6%       | 87.4%  | 불균형이나 사용 가능 |

**두 도서관 모두 적합 성공** (학습/검증 7:3, 층화분리):

| 도서관 | 표본    | 수렴 | Pseudo R² | Test AUC  |
| ------ | ------- | ---- | --------- | --------- |
| 2.28   | 57,897  | ✓    | 0.18      | **0.766** |
| 독산   | 132,966 | ✓    | 0.21      | **0.788** |

핵심 발견 — **공통 신호 + 도서관별 신호가 같이 나온다:**

- **`등록경과`(공통 신호)**: 두 도서관 모두 "오래 꽂혀 있었는데도 안 빌린 책"이 강하게 미대출↑.
  단순 집계로는 못 잡는 **신착 함정을 통제** → 도서관 무관하게 재현되는 정리 후보 기준.
- **`장서연령`(도서관별 신호)**: 2.28은 "오래될수록 미대출↑"인데 독산은 효과 거의 없음.
  → **모델 흠이 아니라 자기참조 모델의 핵심 예측과 부합.** 도서관마다 미대출 조건이 다르니
  처방도 달라야 한다 = "전국 일괄 기준이 아닌 도서관별 진단"이라는 우리 주장의 **데이터 증거**.

> 결론: ②트랙은 "될까?"가 아니라 **"된다 + 우리 논지까지 뒷받침한다"가 확인됨.**
> 노트북: `library_app/notebooks/미대출_로지스틱_분석.ipynb`

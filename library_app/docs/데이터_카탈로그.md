# 도서관정보나루 데이터 카탈로그 (빌드 명세)

> auto-detect 진단 엔진의 설계 기준. "어떤 컬럼이 있으면 어떤 진단을 켤지"를 정의한다.
> 출처: ① 보유 실제 파일(`library_app/data/`) ② data4library Open API 17종.

## 데이터 가족(family)별 정리

### A. 장서 단위 — 한 행 = 책 (우리 도서관)
- **실파일**: `2.28도서관 장서 대출목록` (58,020행)
- **컬럼**: 도서명·저자·출판사·발행년도·ISBN·부가기호·주제분류번호·도서권수·대출건수·등록일자
- **API**: 도서관별 장서/대출 데이터 조회 (`libCode`, `startDt`/`endDt`)
- **auto-detect 신호**: `{도서권수, 대출건수, 주제분류번호}` 존재
- **진단**: 수급 격차(분야/독자대상), 회전율, weeding(발행년도+미대출), 신착(등록일)
- **처방**: 부족 분야 → 수서 / 과잉+저회전 → 정리·큐레이션 / 고회전 → 복본

### B. 시계열 — 월별·연별
- **실파일**: `테마요청 결과파일`(대출월·대출건수, 71개월), `연도별 참여도서관수`(참여연도·신규·누적)
- **API**: 도서관별 대출반납추이 (`libCode`, `type`)
- **auto-detect 신호**: `{날짜/월 컬럼, 대출건수}` 존재
- **진단**: 대출 추세(이동평균·증감), 계절성
- **처방**: 하락 추세 → 프로모션·연계행사

### C. 인기·트렌드 — 순위 목록 (전국 또는 도서관별)
- **실파일**: `BestLoanList`(인기대출 200: 순위·서명·ISBN·ISBN부가기호·KDC·대출건수), `대출급상승도서`(상승폭·기준일순위·전주순위·서명·ISBN), `신착도서`(순위·서명·ISBN·부가기호·KDC·도서권수·도서관수)
- **API**: 인기대출도서(`성별·연령·지역·KDC` 필터), 대출급상승도서, 도서관/지역별 인기대출
- **auto-detect 신호**: `{순위 또는 상승폭, ISBN, 대출건수}` 존재
- **처방 핵심**: **전국 인기도서 ISBN ↔ 우리 소장 ISBN 대조 → 미소장 인기도서 = 수서 추천**
- **파싱 주의**: 메타 12줄 skip / ISBN부가기호 5자리 0-패딩 후 첫자리=독자대상 / 대출건수 콤마 제거

### D. 키워드
- **실파일**: `이달의 키워드`(순위·키워드·Score, 100건)
- **API**: 이달의 키워드(`month`), 도서 키워드(`isbn13`)
- **auto-detect 신호**: `{키워드, Score}` 존재
- **진단/표현**: 사회 관심사 시각화(막대/워드클라우드)

### E. 도서관 마스터 — 한 행 = 도서관
- **실파일**: `참여도서관목록`(도서관명·주소·위도·경도·운영시간·휴관일·도서관코드, 1,598관)
- **API**: 정보공개 도서관 조회, 도서관별 통합정보
- **auto-detect 신호**: `{위도, 경도}` 존재
- **진단**: 접근성(공간 분포 지도 / 운영시간·휴관일 = 시간 접근성)

### F. 도서 상세 — 한 책 심층
- **실파일**: `나미야...`(도서별 이용분석: 대출연월 추세 + 다대출그룹[연령×성별] + 주요키워드 + 함께대출된도서 + 추천도서)
- **API**: 도서별 이용분석(`isbn13`), 마니아/다독자 추천(`isbn13`)
- **구조**: 평평한 표가 아닌 **섹션형** → 전용 파서 필요
- **진단**: 도서 수요 추세, 누가(연령×성별) 읽나
- **처방**: 연관도서 기반 큐레이션·추천

## auto-detect 라우팅 요약

| 데이터에 있으면 | 자동으로 켜지는 진단 |
|---|---|
| 도서권수 + 대출건수 + 주제분류번호 | 장서 수급 진단 + 수서/정리 처방 |
| 날짜/월 + 대출건수 | 대출 추세 진단 |
| 순위/상승폭 + ISBN | (소장과 대조 시) 수서 추천 처방 |
| 위도 + 경도 | 접근성 지도 |
| 키워드 + Score | 관심 키워드 |
| 연령 + 성별 | 이용자 진단 |

→ 사용자는 컬럼을 고르지 않는다. **있는 데이터에 맞춰 진단이 자동으로 등장**한다.
